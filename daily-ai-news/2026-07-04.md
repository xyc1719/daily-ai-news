## 1. VT-WAM：视觉-触觉世界动作模型，打通"感知→世界预测→动作"全链路

**论文**：[VT-WAM: Visual-Tactile World Action Model for Contact-Rich Manipulation](https://arxiv.org/abs/2607.02503)
**作者**：Shuai Tian, Yupeng Zheng, Yuhang Zheng 等（多机构合作，含香港科技大学）

**简述**：提出 VT-WAM，一个融合**视觉与触觉信号**的世界动作模型（World Action Model），直接面向接触丰富（contact-rich）的精细操作任务。不同于传统 VLA 仅依赖视觉，VT-WAM 在"世界模型预测未来状态→动作规划"框架中显式引入触觉模态，使模型能感知力、摩擦、形变等纯视觉难以捕获的物理信息。

**值得关注的原因**：
- **范式级融合**：将 World Model 的预测能力与 Action Model 的执行能力统一，同时引入触觉——这是 VLA 向真正物理交互迈出的关键一步
- **填补研究空白**：当前 VLA/世界模型主要依赖视觉，触觉在具身操作中至关重要却长期被忽视。VT-WAM 提供了将触觉融入世界动作模型的系统方案
- **与你的方向直接相关**：视频分析（视觉感知）+ 具身操作（物理动作）= 多模态世界动作模型，正是三域交叉的典型案例

**技术启发**：触觉可以作为世界模型的"物理先验"，约束视觉预测的一致性——考虑用触觉信号作为视频世界模型的额外监督或正则化项。

## 2. WorldSample：世界模型驱动的闭环真实机器人强化学习

**论文**：[WorldSample: Closed-loop Real-robot RL with World Modelling](https://arxiv.org/abs/2607.02431)
**作者**：Yuquan Xue, Le Xu, Zeyi Liu 等

**简述**：提出 WorldSample 框架，将世界模型（World Model）与**真实机器人闭环 RL** 深度结合。传统世界模型大多在仿真中做规划，到真实机器人上部署面临严重的 sim-to-real gap。WorldSample 通过在真实机器人数据上持续更新世界模型，让 RL 策略可以在"想象"和"真实"之间无缝切换——世界模型提供高效的规划空间，真实交互提供准确的反馈校正。

**值得关注的原因**：
- **闭环范式突破**：解决了"世界模型只在仿真中有用"的核心痛点，给出了真实机器人闭环 RL 的工程化方案
- **与 SimFoundry/SCORE 形成趋势线**：从上周的 Real-to-Sim-to-Real (SCORE) 到本周的真实世界闭环建模，领域正在快速打通仿真与真实
- **可复现性**：框架强调在真实机器人上的部署，论文可能包含硬件设置和实验细节，对实验派研究者参考价值大

**技术启发**：考虑用世界模型生成"反事实"rollout 作为 RL 的数据增强——在世界模型的隐空间中做数据增广，比在像素空间更物理一致。

## 3. Procedural Memory Distillation：Agent 记忆从检索走向可学习技能

**论文**：[Procedural Memory Distillation: Online Reflection for Self-Improving Language Models](https://arxiv.org/abs/2607.01480)
**作者**：Ye Liu, Srijan Bansal, Bo Pang 等（**Salesforce Research**）
**使用模型**：Qwen3-8B, OLMo3-Instruct-7B

**简述**：提出**过程记忆蒸馏（PMD）**，将 Agent 跨 episode 的交互经验蒸馏为可复用的"过程记忆"，并在训练中将其注入策略权重。记忆在三个抽象层次组织：原始轨迹 → 自我反思策略/教训 → 跨问题的高层行为模式。在 Qwen3-8B 上，PMD 相比 SDPO 在 SCIKNOWEVAL 上提升 3.8-5.5%，在 LIVECODEBENCH 上提升 7.9-13.6%。

**值得关注的原因**：
- **记忆范式转变**：从 AutoMem 的"记忆作为可学习技能"到 PMD 的"跨 episode 经验蒸馏"，Agent 记忆正从工程架构（向量检索）演变为**训练方法论**（经验蒸馏→权重更新）
- **训练信号创新**：PMD 的关键贡献在于定义了"什么构成好的经验"的三层抽象——这对 Agent RL 训练中的数据构建有直接指导意义
- **与你上周关注的 OPID 呼应**：OPID 做 On-Policy 技能蒸馏，PMD 做跨 episode 过程蒸馏，都在探索"从 Agent 交互中提取可复用知识"的方法论

**技术启发**：三层记忆抽象（轨迹→反思→模式）可以迁移到具身场景——机器人的操作经验也可以在类似层次结构中被蒸馏和复用。

## 4. SimWorlds：多 Agent 协作生成动态 4D 场景

**论文**：[SimWorlds: A Multi-Agent System for Dynamic 3D Scene Creation](https://arxiv.org/abs/2607.01766)
**作者**：Chunjiang Liu, Xiaoyuan Wang, Haoyu Chen, Yizhou Zhao, **Ming-Hsuan Yang**, László A. Jeni（CMU 等）
**基准**：4DBuildBench

**简述**：提出 SimWorlds，一个多 Agent 系统从文本描述自动生成**动态可编辑 4D 场景**。采用 planner-coder-reviewer 三 Agent 协作工作流：Planner 协调空间布局和物理求解器选择，Coder 生成场景代码（时间序列/摄像机/光照），Reviewer 验证物理一致性。同时发布 4DBuildBench 基准，用于评估 4D 场景生成质量。

**值得关注的原因**：
- **Agent × 世界构建**：多 Agent 协作创建物理仿真环境——这直接触碰了"Agent 自主搭建仿真场景"这一前沿方向
- **与具身智能的结合点**：如果 Agent 能生成高质量 4D 场景，就能为机器人训练提供无限的多样化仿真数据——本质上是一个**Agent 驱动的数据引擎**
- **Ming-Hsuan Yang 组出品**：该组在视觉+AI 领域产出稳定，代码/基准通常会开源

**技术启发**：考虑用 Agent 协作生成具身操作的仿真场景——不同 Agent 分别负责物理参数（重力/摩擦）、物体放置、光照条件，实现大规模自动化数据生成。

## 5. OPINE-World：Agent 通过交互自主学习程序化世界模型

**论文**：[OPINE-World: Programmatic World Modeling with Ontology-error-Prioritized Interactive Exploration](https://arxiv.org/abs/2607.01531)
**作者**：David Courtis, Wenhao Li, Scott Sanner（多伦多大学）
**评测**：ARC-AGI-3（解决 25 个游戏中 20 个）

**简述**：提出 OPINE-World，一个 LLM Agent 通过与环境交互**在线学习以对象为中心的程序化世界模型**。核心创新是耦合两个协作 Agent：执行 Agent 负责探索和行动，模型合成 Agent 负责从观察中归纳世界规则。使用贝叶斯本体误差度量（Ontology Error）引导探索方向，使 Agent 优先探索"当前模型最不确定"的区域。在 ARC-AGI-3 上解决 25 个游戏中 20 个，表现显著优于纯端到端方法。

**值得关注的原因**：
- **Agent 认知 × 世界模型**：Agent **自己学习**世界如何运作，而非使用预训练世界模型——这对 Agent 在新环境的自主适应能力至关重要
- **认知科学启发**：贝叶斯本体误差驱动的探索本质上模拟了人类"发现自己的知识盲区→针对性学习"的认知过程
- **可解释性**：程序化世界模型是符号化的、可读的，避免了黑盒世界模型的可解释性问题

**技术启发**：在具身 Agent 中引入"不确定性驱动的主动探索"——让机器人在操作中自动发现自己对哪些物理交互缺乏准确预测，并优先收集那些场景的数据。
