## 1. VLA-Corrector — 事件驱动的自适应动作时域推理框架

**来源**：浙江大学 OmniAI 组，arXiv:2607.01804，HF 热榜，代码已开源  
**链接**：https://arxiv.org/abs/2607.01804 | https://github.com/ZJU-OmniAI/vla-corrector

**内容简述**：  
现有 VLA 策略普遍采用 action chunk 机制（固定动作窗口的 open-loop 执行），虽然在时序一致性上效率高，但在接触密集的物理交互中，小扰动会在 open-loop 盲区内迅速放大，导致复合误差和任务失败。VLA-Corrector 提出一个**轻量级检测-校正推理框架**：通过 Latent-space Vision Monitor (LVM) 在线对比预测与实际的视觉特征演化，检测到持续偏差时触发截断动作并调用 Online Gradient Guidance (OGG) 进行修正重规划。整个过程**无需修改 VLA 骨干权重**，可在多种 VLA 模型上即插即用。

**值得关注的原因**：
- 这是 **VLA 测试时优化**的第 5 篇连续论文（E-TTS → T²VLA → Guided Action Flow → VLA-Corrector），已形成完整的方法论趋势——不重训模型，仅在推理阶段提升鲁棒性
- "事件驱动的自适应行为"概念非常接近**感知-行动闭环**的核心理念，对具身智能研究者是很好的设计范式
- 开源代码可直接复现，可集成到现有 VLA pipeline 中

**技术启发**：LVM 的隐空间视觉监控思路可以迁移到视频理解任务中——用预测-实际特征偏差作为触发信号，设计事件驱动的视频分析 pipeline。

## 2. Dynamic-SAGE — 视频理解 × Agent 工具合成的双重突破

**来源**：Kurban Intelligence Lab (Texas A&M at Qatar)，arXiv:2607.01469  
**链接**：https://arxiv.org/abs/2607.01469 | https://github.com/KurbanIntelligenceLab/Dynamic-SAGE

**内容简述**：  
Agentic VideoQA 系统在推理时调用工具库中的原子工具，但每个问题都要从零组合原语，推理开销大。Dynamic-SAGE 提出**动态工具合成**：系统在运行中自动合成、验证并持久注册可复用的复合工具——这些工具对后续未见问题同样可用。同时提出了一个**成本-准确度联合审计协议**，将每对系统在六个结果组中分类（联合正确性 + 工具调用变化），用 McNemar 检验和配对 bootstrap 置信区间评估显著性。在 SAGE-Bench 上：accuracy +7.5pp (p < 0.001)，推理轮次和可见工具调用减少约 28%，但 token 使用量增加 34%。

**值得关注的原因**：
- **视频 × Agent 直接交叉**：这是目前少有的将 Agent 工具使用能力直接嵌入视频理解框架的工作
- "工具合成+持久化注册"是一种**Agent 长期记忆**的工程实现，对研究 Agent 记忆机制有启发
- 联合审计协议本身就是一个值得推广的评估方法——accuracy 和 cost 必须联合考虑，这是当前 Agent 评估的盲区
- 代码开源，可复现

**技术启发**：工具合成的思想可以泛化到具身智能——机器人是否也能在执行过程中动态合成"技能原语"并持久化？这就是 ASPIRE 做的事，但 Dynamic-SAGE 的审计协议提供了一种更严谨的评估方法。

## 3. Procedural Memory Distillation — Agent 记忆从检索走向训练

**来源**：Salesforce Research，arXiv:2607.01480  
**链接**：https://arxiv.org/abs/2607.01480

**内容简述**：  
当前 RLVR 训练只使用 episode 级的验证信号更新策略，但 rollout 中更丰富的过程信息（成功策略、失败模式、跨问题行为模式）被丢弃。PMD 将 rollout 信息转化为**三层抽象的程序记忆**：原始轨迹 → 自反思策略/教训 → 跨问题行为模式，全部在线从模型自身轨迹中提取。核心设计是**共进化 (co-evolution)**：策略产生 rollout 更新记忆，记忆塑造监督信号更新策略，两者互相促进。实验结果：Qwen3-8B 上比 SDPO 提升 3.8-5.5%（SCIKNOWEVAL）和 7.9-13.6%（LIVECODEBENCH）；冻结任一组件，性能落后 10%+。

**值得关注的原因**：
- **Agent 记忆范式转变**：从 RAG-based 检索走向可训练的认知技能，训练完成后模型本身即"携带记忆"，无需外部存储
- 共进化机制是理解 Agent 学习和记忆之间关系的新框架——不是简单的"先学再记"或"先记再学"，而是动态相互促进
- Salesforce Research 出品，实验扎实（消融实验证明共进化的核心作用）

**技术启发**：PMD 的三层记忆抽象（轨迹→策略→模式）可以直接借鉴到 VLA 训练中——机器人的操作经验也可以被组织为三层："执行了什么动作" → "什么策略有效/失败"→ "跨任务的通用操作模式"。这是一种**具身 Agent 的程序记忆**设计范式。

## 4. Neuro-Symbolic Safety Guidance for VLA — 预测性碰撞避免

**来源**：University of Central Florida，arXiv:2607.01378  
**链接**：https://arxiv.org/abs/2607.01378 | 项目页面：https://willenglish.tech/SafetyGuidedFlowMatching/

**内容简述**：  
现有 VLA 安全措施只能防止机器人**下一步**动作导致的碰撞。本文提出在流匹配 VLA 的去噪过程中**插入符号约束**，将安全执行形式化为最小范数约束优化问题，在预测轨迹的迭代去噪过程中即时修正安全违规。这使得系统能够在碰撞变得不可避免之前就预测性地避开。SafeLIBERO 基准上：碰撞避免 82.8%（+6.3%），任务成功率 81.6%（+19.8%），长程任务上增益最为显著。

**值得关注的原因**：
- **神经-符号 AI × 具身智能**的干净结合——不是两套独立系统，而是在神经生成过程中穿插符号约束
- "预测性"而非"反应式"的安全机制，对真实机器人部署至关重要
- 流匹配 (Flow Matching) 作为 VLA 的动作生成范式正在扩散——此前 Guided Action Flow 也用流匹配做推理优化，本文进一步将安全约束嵌入同一范式

**技术启发**：约束流匹配的思路可以推广——不仅约束"安全"，是否也可以约束"物理合理性"（如作用力-反作用力）？这将直接连通世界模型的物理一致性研究。

## 5. RLVR for Tool-Use Agents — 纯轨迹奖励训练工具调用

**来源**：独立研究者（Karthikeya Vissa 等），arXiv:2607.01465  
**链接**：https://arxiv.org/abs/2607.01465

**内容简述**：  
构建 5 个合成环境模拟 Jira REST v3 和 Confluence v2 API（schema 级保真度），用 RLVR（GRPO）训练小模型（Qwen3-1.7B / 3.5-4B）的工具调用能力。奖励完全来自工具调用轨迹（无人工标注、无学习 judge、无 live API）：检查参数完整性、调用顺序正确性、结果一致性。4B 模型在 Confluence 页面创建任务上从 0.35→1.00，4 个非退化场景中奖励均收敛到 0.95-1.00。

**值得关注的原因**：
- 证明了 **RL 训练 agent 工具调用**的可行性：不需要 SFT 数据，纯靠环境奖励即可学会正确调用 API
- 合成环境+可验证奖励的范式可以被复用——任何有 schema 定义的 API 都可以构建类似训练环境
- 论文坦率指出了局限性（可验证奖励的手工构造不可大规模扩展），这种诚实为后续研究指明了方向

**技术启发**："合成环境 → RL 训练 → Agent 技能"的范式与具身智能中的 Real-to-Sim-to-Sim-Real 形成有趣的对照——是否可以用类似方法为具身 Agent 构建 API 化的操作环境，训练其工具调用（操作原语）能力？
