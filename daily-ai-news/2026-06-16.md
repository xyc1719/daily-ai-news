## 1. 智源发布全球首个通用世界基座模型 「悟界·Physis-v0.1」

**来源**：北京智源人工智能研究院（BAAI），2026 智源大会（6 月 12-14 日）

**事件简述**：
智源研究院在第八届北京智源大会上正式发布 **悟界·Physis-v0.1**，宣称这是全球首个通用世界基座模型。其核心思路是从"预测下一个 token"转向**"预测下一个物理状态"**，通过专属物理状态编码器将视频、深度 RGB、3D 点云、力触反馈等多模态信息统一转化为标准化隐空间物理状态。模型具备四大特性：**物理正确、动作因果可溯、长程一致、通用泛化**，已支持 50+ 复杂物理场景的长程推理。同时发布的配套具身大脑 **悟界·RoboBrainOrca** 基于 Physis 实现"想-看-动"三位一体闭环，可支撑机器人在物流、酒店等真实环境中的长期自主作业。大会同步发布了多模态神经科学大模型 Brainμ1.0、多模态统一模型 Emu3.5，以及 FlagOS 2.1 软硬件生态。

**值得关注的原因**：
- **范式级突破**：这是业界首次系统性提出以"物理状态预测"替代"token 预测"作为世界模型的训练目标。对比 Genie、Sora 等以像素/视频生成为导向的世界模型路线，Physis 的隐空间物理表征路径在**动作因果推理**和**长程一致性**上有理论优势。
- **三域交叉的绝佳锚点**：Physis 同时覆盖视频理解（多模态输入）、具身智能（机器人操作/导航）和 Agent 决策（物理推理 + 规划的闭环），是研究生构建跨域研究思路的天然参照系。
- **国产开源生态信号**：智源延续"悟道"系列的开源传统，虽然 Physis 具体开源协议尚未公布，但 FlagOS 2.1 的发布意味着国产芯片适配栈正在成型，未来复现和部署的门槛可能大幅降低。
- **与 NVIDIA Cosmos 3 形成双雄格局**：Cosmos 3（6 月初发布）走全模态开放路线，Physis 走物理隐空间路线，两者的对比分析本身就是一篇好论文的选题。

**技术启发**：
> 物理隐空间表征 + VLA（视觉-语言-动作）模型的结合可能是下一个爆发点——能否用 Physis 的物理编码器替换 VLA 中简单的视觉 backbone，让机器人的动作预测具有物理因果一致性？

## 2. StreamMemBench — Agent 记忆的流式评测新范式（首次用自我中心视频测记忆）

**来源**：arXiv:2606.14571（2026-06-12 提交），复旦大学等

**事件简述**：
现有 Agent 记忆评测通常孤立测试对话召回率或任务改进，忽略了从**流式观察到后续辅助**的完整轨迹。本文提出 **StreamMemBench**，基于 EgoLife 自我中心视频数据流，围绕每个证据锚点构建**两步任务序列**：第一步测试证据的即时利用，第二步测试反馈和交互经验是否在后续任务中被复用。设计了四个独立诊断指标（证据召回、初始证据利用、反馈融入、后续复用），在两个 backbone 上对 8 种记忆系统全面评测。核心发现：当前系统即使证据已被存储，也往往无法有效利用；即使局部融入了反馈，也无法将其转化为稳定的后续行为。

**值得关注的原因**：
- **视频 + Agent 记忆的深度交叉**：StreamMemBench 使用自我中心视频流（egocentric video）作为 Agent 的记忆输入源，这直接桥接了视频理解（时序动作定位、事件检测）与 Agent 记忆管理两个领域。对同时关注视频分析和 Agent 的研究生来说，这是一个极佳的结合点。
- **暴露的关键研究空白**：论文揭示了当前记忆系统的核心问题——"存储 ≠ 利用"、"局部融入 ≠ 全局行为改进"。这意味着**记忆推理层（memory reasoning layer）**是一个亟待填补的研究方向，远超简单的 RAG/向量检索范式。
- **可复现性强**：基准已公开发布，评测协议标准化，便于直接对比和改进。

**技术启发**：
> Agent 记忆系统需要一个"记忆推理层"——不只是检索相关记忆，还要对检索到的记忆做时序因果推理，判断"这段过去的观察在当前任务中是否真的有用"——这本质上是一个视频时序定位 + 因果推理的联合问题。

**参考链接**：
- arXiv: https://arxiv.org/abs/2606.14571
- PaperReading: https://paperreading.club/page?id=416572

## 3. HarnessX — Agent 运行时框架的"可演化工厂"，多智能体引擎驱动自动改进

**来源**：arXiv:2606.14249（2026-06-15 新提交），Darwin-Agent 团队；已开源 GitHub

**事件简述**：
HarnessX 提出 **Agent = Model + Harness** 的可组合架构，将 prompt、工具、记忆、控制流等运行时组件类型化为可替换原语，通过**替换代数（substitution algebra）**组装。引入 **AEGIS**——一个 trace 驱动的多智能体演化引擎，基于操作镜像持续发现更优的 harness 配置。执行轨迹同时转化为 harness 更新和模型训练信号，形成双闭环。在 ALFWorld、GAIA、WebShop、τ³-Bench、SWE-bench Verified 五个基准上，平均提升 **14.5%**，最高 **+44.0%**。

**值得关注的原因**：
- **Agent 工程的系统化方法论**：不再靠手工调 prompt，而是将 Agent 的"外壳"（harness）本身作为优化对象，通过多智能体自动搜索最优配置。这对工程落地有直接参考价值。
- **与具身智能的结构类比**：HarnessX 的"操作镜像 + 轨迹反馈"架构与具身智能中的"世界模型 + 策略优化"闭环在结构上高度相似——可以把 harness 看作 Agent 在数字世界的"身体"，AEGIS 是它的"学习引擎"。这种类比可能催生统一的跨域框架设计。
- **开源可验证**：GitHub 已开源（github.com/Darwin-Agent/HarnessX），可直接复现和扩展。

**技术启发**：
> 如果 HarnessX 的"操作镜像"概念扩展到具身场景——即用仿真环境作为 harness 的操作镜像，通过 AEGIS 自动搜索最优的感知-规划-控制流水线配置——就是一个视频/Agent/具身三域统一的研究框架。

**参考链接**：
- arXiv: https://arxiv.org/abs/2606.14249
- GitHub: https://github.com/Darwin-Agent/HarnessX
- HuggingFace Papers: https://huggingface.co/papers/2606.14249

## 4. RefGRPO — 弥合 Agent 的"反思鸿沟"，零成本校准反思能力

**来源**：arXiv:2606.14211（2026-06-15 新提交）

**事件简述**：
本文发现并定义了 **"反思鸿沟"（Reflection Gap）**——LLM Agent 在观察到环境反馈后仍会错误评估自身表现（即使答案正确也倾向说不正确），而标准 RL 因信用分配不匹配无法有效修复。提出 **RefGRPO**：通过对比 Agent 自我反思与实际结果计算"免费校准奖励"（无需额外奖励模型或外部标注），配合动态系数调度策略。在 5 个 text-to-SQL 基准上，欠自信率从 **44.4% 降至 7.7%**，任务准确率从 75.1% 提升至 76.5%。校准后的反思使 Agent 可充当自身验证器，支持基于反思伪奖励的自我改进。

**值得关注的原因**：
- **Agent 训练的新方向**：RL 用于 Agent 训练是当前热点（DeepSeek、Kimi 等都在探索），但反思能力的训练一直缺乏有效方法。RefGRPO 的"免费校准"思路优雅且实用，不需要额外标注成本。
- **对具身 Agent 的潜在价值**：具身场景中，机器人也需要在执行动作后评估"我刚才做对了吗"——RefGRPO 的反思校准方法可直接迁移到 VLA 模型的 RL 训练中，缓解具身场景更严重的信用分配问题。
- **方法简洁，可快速复现**：只需在标准 GRPO 基础上加一个校准项，适合快速集成到现有 RL pipeline 中验证。

**技术启发**：
> 将 RefGRPO 的反思校准机制引入 VLA 训练——让机器人在执行操作后不仅学习"动作是否正确"，还学习"对自己动作的评估是否准确"——这可能是提升具身 Agent 鲁棒性的低成本路径。

**参考链接**：
- arXiv: https://arxiv.org/abs/2606.14211

## 5. COMET — 对象中心因果世界模型 + MCTS 实现高效机器人操作规划

**来源**：arXiv:2606.14418（2026-06-15 新提交）

**事件简述**：
提出 **COMET（Causal Object-centric Model for Efficient Tree search）**，将**对象中心表示**（slot-structured object-centric world model）、**因果推理**（symbolic regression 提取因果图）与 **MCTS（蒙特卡洛树搜索）**三者结合，用于机器人操作任务的模型式规划。在 ManiSkill、Robosuite 等基准上，COMET 在早期训练阶段（样本效率）显著优于无模型和标准模型式 RL 方法。

**值得关注的原因**：
- **世界模型 + 搜索的具身落地**：AlphaGo 式的 MCTS 规划在具身场景一直面临状态空间巨大的问题。COMET 通过对象中心表示有效压缩了状态空间，使得树搜索在机器人操作中变得可行——这正是"从棋盘到物理世界"的关键一步。
- **因果推理的引入**：在对象中心表示的基础上进一步提取因果图，使模型不仅能预测"发生了什么"，还能理解"为什么发生"——这对需要长程推理的复杂操作任务（如多步装配）至关重要。
- **与视频理解的连接点**：对象中心表示（slot attention 类方法）本身是无监督视频理解中的重要范式，COMET 将其扩展为世界模型用于规划，为"从视频中学习世界模型然后做规划"提供了完整的技术链路。

**技术启发**：
> 能否从第一人称视频中无监督地学习对象中心世界模型，然后直接用于下游的 MCTS 操作规划？这将把视频理解、世界模型学习和具身操作串联成一条端到端的研究 pipeline。

**参考链接**：
- arXiv: https://arxiv.org/abs/2606.14418
