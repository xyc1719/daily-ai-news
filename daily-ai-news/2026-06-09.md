## 1. 🎬 Thinking with Video：视频生成作为统一多模态推理范式

 **来源** ：CVPR 2026 | 论文：*Thinking with Video: Video Generation as a Promising Multimodal Reasoning Paradigm*
 **链接** ：[项目主页](https://thinking-with-video.github.io/) | [CVPR Open Access](https://openaccess.thecvf.com/content/CVPR2026/html/Tong_Thinking_with_Video_Video_Generation_as_a_Promising_Multimodal_Reasoning_CVPR_2026_paper.html) | [GitHub](https://github.com/tongjingqi/Thinking-with-Video)

 **事件简述** ：提出全新范式——用视频生成模型（Sora-2）以视频帧作为统一介质进行多模态推理，跳出"文本CoT"和"图像推理"框架。构建了 VideoThinkBench 基准（视觉+文本任务），实验表明 Sora-2 在眼球谜题上 **超越 GPT-5 达 10%** ，MATH 上达 92%（!!），MMMU 上 69.2%。

 **值得关注的原因** ：这是一篇范式级工作。核心 insight 极具启发性——视频帧天然融合视觉+文本+时序，可能成为"统一多模态推理语言"。对视频分析方向的同学来说，这意味着： **视频生成模型不只是"生成器"，而是潜在的通用推理器** 。如果这个方向成立，video generation → video understanding → agent reasoning 会连成一条线。

 **技术启发** ：视频生成模型的内部表征能否蒸馏出来作为视觉推理的通用 backbone？类似"用 diffusion model 做分类"的思路但拓展到时序推理。

## 2. 📝 Watch, Remember, Reason：MLLM 视频理解综述

 **来源** ：arXiv 2606.07433（2026-06-05 提交）| 15 位作者，跨机构合作
 **链接** ：[arXiv](https://arxiv.org/abs/2606.07433) | [GitHub Awesome List](https://github.com/marinero4972/Awesome-HumanView-VideoUnderstanding)

 **事件简述** ：从人类视角出发，将基于 MLLM 的视频理解系统化为 **观看（Watching）→ 记忆（Remembering）→ 推理（Reasoning）** 三能力框架。覆盖细粒度/视听/高效感知、离线与流式记忆、纯文本与视频思维推理，并系统梳理了自我中心、体育、教学、医学、叙事视频等应用场景。

 **值得关注的原因** ：这篇综述的框架直接映射到你关注的三个方向的交叉点——"观看"对应视频分析，"记忆"对应 Agent 的记忆模块，"推理"是两者的桥梁。特别值得注意的是对 **流式理解（streaming understanding）**和**自我中心视频（egocentric）** 的覆盖，这正好连接具身智能中的第一人称感知。作为研究生，这篇综述可以用来快速建立该交叉领域的全景地图。

 **技术启发** ：目前的 Agent 记忆系统（如 SubtleMemory 揭示的）还很弱，视频 MLLM 的记忆机制能否迁移给通用 Agent？"视频记忆"的研究可能比"文本记忆"更接近人类认知。

## 3. 🤖 AnchorWorld：具身第一人称世界模拟 + 自演化定制

 **来源** ：arXiv 2606.07326（2026-06-05 提交）| 多位来自腾讯+高校的作者
 **链接** ：[arXiv](https://arxiv.org/abs/2606.07326) | [HTML](https://arxiv.org/html/2606.07326v1)

 **事件简述** ：提出 AnchorWorld 框架，以 **3D 人体运动**作为交互驱动，引入 **外源视点** （exogenous views）解决自我中心视角下的身体遮挡问题，并通过在世界坐标系中定义**锚定视图 + 文本描述**实现场景的自演化定制。实验显示在交互完整性和几何一致性上显著超越基线。

 **值得关注的原因** ：这篇直接打在 **视频 + 具身 + 世界模型** 的交叉点上。两个关键创新：(1) 外源视点辅助训练——这是一个非常实在的工程思路，解决了 egocentric 感知的盲区问题；(2) 锚定视图的文本驱动演化——让世界模拟变得可控，这对具身 Agent 的训练数据生成有直接价值。如果后续能开源，是做具身仿真+视频生成的优质 baseline。

 **技术启发** ："外源视点辅助 egocentric 感知"这个思路能否推广？比如具身 Agent 在真实环境中部署时，是否可以用第三方摄像头作为辅助感知源？

## 4. 🧠 Socratic-SWE：Agent 从自己的"解题草稿"中自我进化

 **来源** ：arXiv 2606.07412（2026-06-05 提交）| 8 位作者，Under Review
 **链接** ：[arXiv](https://arxiv.org/abs/2606.07412) | [HuggingFace](https://huggingface.co/papers/2606.07412)

 **事件简述** ：提出闭环自进化框架——从 Agent 的历史求解轨迹中蒸馏 **结构化技能** （失败模式 + 修复模式），技能反过来指导生成针对性训练任务，再通过执行验证 + 梯度对齐奖励筛选高质量任务，训练后的 Agent 产生新轨迹，形成自我迭代。SWE-bench Verified 达到  **50.40%** （三轮迭代），持续优于同算力自进化基线。

 **值得关注的原因** ：这篇对 Agent 方向的研究生有一个核心理念启发： **Agent 的"失败"本身是最有价值的训练信号** 。传统思路是"更好的 prompt / 更强的模型 / 更多的工具"，而 Socratic-SWE 把 Agent 自身的求解轨迹变成了可迭代优化的资产。这种"自我反思→技能提炼→针对训练→能力提升"的闭环，比 RLHF 或 SFT 的 offline 训练更接近人类学习。

 **技术启发** ：这个框架能否泛化到非代码领域？比如具身 Agent 在仿真中的操作轨迹，是否可以类似地提炼为"操作技能"，用于生成针对性的训练场景？

## 5. 🦿 LIMMT：人形机器人运动跟踪——少即是多

 **来源** ：ICML 2026（已接收）| arXiv 2606.06953 | 作者来自多机构
 **链接** ：[arXiv](https://arxiv.org/abs/2606.06953) | [项目主页](https://giraffeguan.github.io/limmt/) | [GitHub](https://github.com/GalaxyGeneralRobotics/Humanoid-GPT/blob/main/projects/LIMMT.md)

 **事件简述** ：首次对物理仿真人形机器人运动跟踪进行系统性的 **数据中心性研究** 。从物理可行性、多样性、复杂性三个维度定义运动数据质量，证明仅用 **AMASS 3% 的数据**训练即可超越全量数据性能。还构建了 GQS pipeline 用于网络源动捕数据的清洗。

 **值得关注的原因** ：这条对做具身智能的同学有一个反直觉的启示—— **数据质量 >> 数据规模** 。在大家都疯狂堆数据（百万轨迹、TB 级视频）的当下，这篇用严谨实验证明：精心筛选的极小训练集可以打败海量噪声数据。对资源有限的高校实验室尤其有参考价值：与其花大价钱采集海量数据，不如把精力放在数据质量筛选上。

 **技术启发** ：视频分析领域是否也存在类似的"数据质量悖论"？视频理解模型的训练数据筛选——是否也能用物理可行性 + 多样性 + 复杂性的框架来重新定义？
