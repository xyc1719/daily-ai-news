## 1. 🏛️ WAIC 2026 上海开幕：具身智能与 Agent 成为绝对主角

**来源：** WAIC 2026 官方 | 2026-07-17 ~ 07-20 | 上海世博展览馆

**事件简述：**
2026 世界人工智能大会（WAIC 2026）今日正式开幕，展览面积首次突破 10 万平方米，1100+ 企业参展，超 300 款新品全球首发。本届大会核心看点：

- **蚂蚁灵波"全栈大脑 2.0"**：获评大会"十大镇馆之宝"。核心是 **LingBot-VLA 2.0**——基于 6 万小时真实数据预训练的跨本体具身基座模型，已适配 17 家厂商 20+ 种机器人构型，实现"一脑多机"。同步发布 **LingBot-VA 2.0**（行业首个具身原生世界动作模型）和 **LingBot-World 2.0**。现场展示的智慧药房方案已在国大药房落地，3 台不同构型机器人 90 秒内完成接单→取药→打包全流程。
- **商汤 Agent Seko**：全球首个创编一体的 AI 视频创作 Agent，打通剧本构思→角色设定→分镜→视频合成→配音→配乐全流程。
- **百度"百度搭子"**：通用智能体，自然语言驱动即可完成文件处理、数据分析、PPT 制作等复杂工作。
- **智元机器人**：将发布远征 A3 Ultra 全尺寸人形机器人等新品矩阵。
- **加速进化 Booster T2**：机器人世界杯卫冕冠军，现场点球大战+跑酷。

**值得关注的原因：**
- **产业拐点信号**：本届 WAIC 明确传递"2026 是 Agent 实干元年"的信号——从去年的"能聊"全面转向"能干"。对于研究生而言，这意味着 Agent 和具身智能的研究选题需要更关注**真实场景的可部署性**（延迟、跨本体泛化、长时稳定性），而非仅追求 benchmark 指标。
- **技术路线收敛**：蚂蚁灵波"全栈大脑 2.0"的三层架构（感知层→生成层→执行层）代表了一种日渐清晰的具身智能系统设计范式，值得作为研究 reference architecture。
- **视频+Agent 交叉**：商汤 Agent Seko 将视频创作全流程 Agent 化，展示了视频生成如何从"模型能力"升级为"Agent 系统能力"——这对视频分析方向的 Agent 设计有直接启发。

**技术启发：**
> LingBot 的"一脑多机"架构能否启发视频分析领域的"一模型多任务"设计——单一视频理解模型通过不同的"解码头"同时支持分类、检测、描述、检索等多种下游任务？

## 2. 🧮 Ring-Zero：万亿参数 Zero RL 训练涌现推理能力

**来源：** inclusionAI | arXiv:2607.12395 | 2026-07-14 | HuggingFace Daily Papers 79🔥

**事件简述：**
inclusionAI 团队发布 Ring-Zero，首次将 Zero RL（无需人类标注 CoT 数据的强化学习）扩展到 **1 万亿参数 MoE 模型**（63B active parameters）。使用 320 张 H200 GPU，基于 Megatron + SGLang + Areal 框架完成训练。核心发现：

1. **Scaling Law 验证**：1T 模型在样本效率上远超 104B 模型，且性能上限显著更高——AIME 2026 Pass@1 达 84.2%（第一阶段），后续阶段在多个数学基准上进入 90%+。
2. **两阶段训练动态**："discovery phase"（探索新推理行为）→"sharpening phase"（在已发现空间中精炼策略），这与 Agent 系统的"探索-利用"平衡问题高度对应。
3. **涌现行为**：模型自发产生结构化格式化、自验证、并行推理、拟人推理痕迹、"context anxiety"（对上下文使用的谨慎）等高级认知行为，无需人工设计启发式规则。
4. **分层推理预算**：模型学会了在 4K/16K/64K token 预算下自适应调整推理深度——浅层问题不浪费算力。

**值得关注的原因：**
- **对 Agent 训练的启发**：Zero RL 的"discovery→sharpening"两阶段动态为 Agent 的策略训练提供了新的思路——也许 Agent 的工具使用和规划能力也需要先"粗放探索"再"精细打磨"，而非一次性 SFT。
- **涌现行为的工程价值**：模型自发产生的"context anxiety"行为说明长上下文管理是推理能力的自然副产品，这对设计长时视频理解 Agent 的记忆管理机制有参考意义。
- **研究空白**：Zero RL 目前高度依赖可验证奖励（数学题有标准答案），如何将其扩展到"无标准答案"的具身操作和视频理解任务是重大开放问题。

**技术启发：**
> Ring-Zero 的分层推理预算（4K/16K/64K）思路能否用于视频理解——让模型根据视频复杂度自适应地分配计算（简单场景秒级理解，复杂场景深度推理）？

## 3. 📱 PalmClaw：手机端原生 Agent 框架，比 GUI 方案快 20 倍

**来源：** 香港理工大学 × ModalityDance | arXiv:2607.13027 | 2026-07-15 | HuggingFace Daily Papers 1.12k🔥（日榜第一）

**事件简述：**
PalmClaw 是一个直接在 Android 手机上运行的**原生 Agent 框架**。它的核心设计哲学是：Agent 不应该通过截图→找按钮→点击的方式操作手机（GUI 路线），而应该直接调用手机的设备工具（Device Tools）——文件系统、传感器、应用 API 等。关键数据：

- **任务成功率 97.1%**（最强 GUI 基线 87.1%，相对提升 11.5%）
- **任务完成时间减少 94.9%**（从分钟级降到秒级）
- 使用 DeepSeek-V4-Flash 作为推理后端
- AGPLv3 开源，安装包可直接下载，设置只需 2 分钟

框架组件全部在手机端运行：会话管理、记忆、技能库、设备工具、Agent 循环。LLM 推理通过远程 API，但上下文管理和工具执行完全本地化。

**值得关注的原因：**
- **范式批判**：PalmClaw 用数据证明"GUI Agent 是过渡方案，不是终点"。手机如此，机器人、智能家居、车载系统同样如此——只要存在结构化 API 或工具接口，就不应该通过像素级操作来驱动。这对 Agent 研究的方法论选择有直接影响。
- **对视频+Agent 交叉的启发**：视频理解 Agent 同样面临"看画面操作" vs "调用结构化感知工具"的选择。例如，与其让 Agent 逐帧分析视频来定位事件，不如让 Agent 调用预训练的视频事件检测 API 并基于结果做高层推理。
- **端侧 Agent 趋势**：PalmClaw 证明了端侧 Agent 在工程上已经可行，后续研究方向将转向端侧模型的推理能力提升和端云协同。

**技术启发：**
> 视频理解 Agent 能否借鉴 PalmClaw 的"设备工具"思路——将视频分析的基础能力（目标检测、动作识别、场景分割等）封装为可调用的 **Video Tools**，Agent 只做高层任务规划和工具编排？

🔗 [论文: arxiv.org/abs/2607.13027](https://arxiv.org/abs/2607.13027) | [代码: github.com/ModalityDance/PalmClaw](https://github.com/ModalityDance/PalmClaw)

## 4. ⚡ GigaWorld-Policy-0.5：更快更强的世界动作模型，Agent 自动调参

**来源：** 极佳视界（GigaAI）× 清华大学 | arXiv:2607.13960 | 2026-07-15 | 开源

**事件简述：**
极佳视界联合清华发布 GigaWorld-Policy-0.5，在上一代 GigaWorld-Policy 基础上实现三大升级：

1. **混合预训练策略**：交替进行 Action-Conditioned World Modeling（AC-WM）和 WAM 训练，强化视觉动态与机器人动作之间的耦合，提升动作表征的可迁移性。
2. **Mixture-of-Transformers 架构**：将视觉动态建模和动作生成分离为不同专家，推理时仅激活动作专家——在 RTX 4090 上推理延迟仅 **85ms**，比上一代提速 3 倍。
3. **Agent-based AutoResearch**：使用 Agent 自动搜索训练配置和超参数，系统性地找到最优实验设置，减少人工调参开销。

模型、代码、技术报告全部开源。

**值得关注的原因：**
- **架构创新**：MoE 分离视觉与动作专家的设计很精巧——训练时利用视觉动态作为密集监督，推理时丢弃视觉生成只做动作解码。这为其他需要"训练用全模态、推理用轻量模态"的场景提供了可复用的架构范式。
- **AutoResearch 信号**：用 Agent 自动化科研流程（超参搜索、实验设计）本身就是一个趋势信号——如果你做 Agent 研究，"AI for AI Research"可能是一个高价值方向。
- **工程实用性**：85ms 推理延迟使世界动作模型真正具备在真实机器人上闭环部署的可行性，这比在仿真环境中刷指标更有工业价值。

**技术启发：**
> MoE 的"视觉-动作分离"思路能否用于视频理解模型——训练时融合多模态（RGB+光流+音频），推理时根据任务只激活所需模态专家？

## 5. 👁️ Vinci2 + EgoServe：第一人称视频从"被动问答"走向"主动辅助"

**来源：** 东京大学 × 上海 AI Lab | ECCV 2026 | arXiv:2607.11523 | 2026-07-14

**事件简述：**
Vinci 系统升级为 Vinci2，将第一人称（egocentric）视频理解从**被动问答**推向**主动辅助**。核心贡献：

- **EgoServe 基准**：首个大规模主动辅助评测集，3000+ 服务实例，覆盖 4 个时序记忆跨度（即时安全警报→短期任务提醒→情节式辅助→长期习惯教练）和 10 个服务类别。
- **EgoMemo Agent**：免训练的 memory-augmented Agent，维护三种互补记忆——多尺度时序摘要、语义知识图谱、视觉嵌入档案。在每个时间步通过检索增强推理判断"是否应该主动介入"。
- 关键设计：主动辅助被重新定义为**上下文依赖的决策问题**——Agent 不仅要感知"正在发生什么"，还要基于累积的时间上下文判断"此时介入是否合适"。这对避免"过度打扰"至关重要。

**值得关注的原因：**
- **视频理解的新范式**：传统视频理解任务是"给定视频+问题→输出答案"，Vinci2 提出"持续流式视频→主动判断何时输出什么"。这对视频分析的研究问题定义有根本性冲击——任务不再是被动的，而是主动的、情境感知的。
- **三记忆架构**：时序摘要+知识图谱+视觉嵌入的互补记忆设计与 Agent 记忆研究（如 MemGPT、NapMem）高度呼应，但 Vinci2 将其落地到了连续视频流场景——这对视频+Agent 交叉研究是极好的参考设计。
- **研究空白**：EgoMemo 是 training-free 的，意味着记忆管理和主动决策逻辑完全通过提示工程实现。如何将这种能力内化到模型权重中（通过训练），是一个明确的研究方向。

**技术启发：**
> EgoMemo 的"三记忆+检索增强推理"架构能否直接迁移到通用视频理解 Agent——让 Agent 在持续观看视频流时自动判断"哪些时刻值得关注并主动报告"？
