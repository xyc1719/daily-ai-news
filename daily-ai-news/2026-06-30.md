## 1. SimFoundry — 从视频零样本构建仿真场景，打通"视频→具身"数据瓶颈

**arXiv**: [2606.28276](https://arxiv.org/abs/2606.28276) | **机构**：NVIDIA / Stanford（含 Li Fei-Fei、Yuke Zhu、Ajay Mandlekar）| **HuggingFace 热榜**

**内容**：提出 SimFoundry，一个模块化、自动化的零样本 real-to-sim 场景构建系统 — **输入一段真实场景视频，输出可直接用于机器人策略训练的仿真数字孪生**。系统支持物体级、场景级、分布级三种"数字表亲"（digital cousins）生成，在 7 个操作任务、5 种策略架构上进行验证，仿真评估与真实性能的 Pearson 相关系数高达 **0.911**。使用"表亲"数据训练的策略在真实世界成功率提升 **17-40%**。

**值得关注**：
- 这是目前**视频分析 + 具身智能 + 仿真**三者最直接的交叉成果。传统 sim-to-real 依赖人工建模，SimFoundry 用视频自动完成场景重建+多样化，将"拍一段视频就能训练机器人"推进了一大步。
- 0.911 的仿真-真实相关性意味着仿真评估的可靠性大幅提升，这对无法频繁做真实实验的研究生尤为重要。
- **NVIDIA + Stanford 强组出品，开源可期**，是视频分析方向切入具身智能的绝佳切入点。

**技术启发**：视频理解不限于语义标签——将视觉重建与物理交互能力结合，可能是视频领域下一个高价值出口。可以思考：如何用视频生成模型（如 video diffusion）来替代传统仿真器做策略评估？

## 2. PhysisForcing — 物理强化的视频世界模型，让生成视频真正"可模拟"

**arXiv**: [2606.28128](https://arxiv.org/abs/2606.28128) | **机构**：NVIDIA | **HuggingFace 热榜 #1**

**内容**：提出 PhysisForcing，一种在视频生成训练中注入层次化物理约束的训练框架。通过**像素级轨迹对齐损失**（pixel-level trajectory alignment）和**语义级关系对齐损失**（semantic-level relation alignment），强制生成视频遵守物理规律。在 R-Bench 上将 Wan2.2 基础模型提升 **22.3%**；作为世界模型在 WorldArena 闭环控制中成功率从 16.0% 提升至 **24.0%**（+50% 相对提升）。

**值得关注**：
- 这是"视频生成作为世界模拟器"路线的关键补充 — **不是生成更"好看"的视频，而是生成更"物理正确"的视频**。当前 video diffusion 模型用于机器人策略评估时经常出现物理不一致（物体悬浮、碰撞错误等），PhysisForcing 直接对症下药。
- 方法具有通用性：可插入任意 video generation backbone，对研究视频世界模型的研究生是直接可用的工具。
- 代表了一个重要方向：**从"视觉逼真"到"物理逼真"的范式切换**。

**技术启发**：考虑将物理先验（如牛顿力学、碰撞检测）作为可微损失函数嵌入到视频生成/理解模型训练中，而非仅依赖数据驱动。这个思路也可拓展到 VLA 模型的 action head 设计。

## 3. SCORE — "实→虚→实"范式突破：不在真实世界也能提升真实世界策略

**arXiv**: [2606.27475](https://arxiv.org/abs/2606.27475) | **机构**：未公开（新投稿排行 cs.RO #1）

**内容**：提出 SCORE（Support-Constrained RL），一种全新的 **Real-to-Sim-to-Real** 范式：在仿真中用 RL 改进策略时，通过 flow steering 约束到真实数据预训练策略的支撑集，确保改进是"安全的"。在 8 个灵巧多指操作任务上，平均成功率从 **37.8% → 89.9%**（最佳基线仅 59.5%）。**完全不需要额外的真实世界交互即可提升真实部署性能**。

**值得关注**：
- 这是 sim-to-real 领域的一次范式级突破。传统 wisdom 认为"在仿真里学得再好，上了真机也会掉性能"；SCORE 证明了**如果你把 RL 约束在真实数据预训练策略的"邻域"内，仿真进步可以无损迁移**。
- 对资源有限的研究生课题组极其实用：不需要大量真机时间，就能持续改进策略。
- 方法框架具有通用性，理论上可以适配任何行为克隆预训练 + RL 微调的 pipeline。

**技术启发**：支撑集约束的思路是否可以拓展到 VLA 模型的后训练？即：用少量高质量真机数据划定"安全策略空间"，然后在仿真中大规模 RL 在这个空间内搜索最优策略。

## 4. DMV-Bench — 多模态 Agent 的视觉记忆到底怎么样？首个交互式评测来了

**arXiv**: [2606.27499](https://arxiv.org/abs/2606.27499) | **领域**：Agent + 视频理解

**内容**：推出 DMV-Bench，首个针对多模态 Agent **视觉记忆**的交互式基准。基于 1000+ 家电家具电商商品目录，构建长时交互式购物视频场景，要求 Agent 在观看视频后回答关于物品属性、位置、变化的记忆问题。评测覆盖了现有主流多模态模型在 visual memory retention 上的显著差距，揭示了"Agent 看视频就忘"的系统性问题。

**值得关注**：
- 这是**视频分析 + Agent 交叉领域的最直接评测**。当前多模态 Agent 普遍缺乏长时视觉记忆能力（类比 LLM 的上下文窗口问题），DMV-Bench 提供了首个标准化衡量工具。
- 对做视频理解的你来说，这是一个天然的"下游任务"方向 — 你可以设计更好的视觉记忆编码/检索机制来刷榜。
- 视觉记忆是 Agent 在真实环境中长时间运行的关键能力（例如：家用机器人需要记住"我 10 分钟前把杯子放在哪了"）。

**技术启发**：视觉记忆≠视频理解。或许需要设计专门的 episodic visual memory 模块（如 video token summarization + keyframe retrieval），而非依赖 LLM 的隐式记忆。

## 5. Agent-Native Immune System — Agent 安全不只是 prompt 加固，需要"免疫系统"

**arXiv**: [2606.28270](https://arxiv.org/abs/2606.28270) | **领域**：Agent 安全

**内容**：提出 Agent-Native Immune System (ANIS)，首个**生物启发的 Agent 内生安全架构**。将 Agent 面临的安全威胁系统化为：工具使用操纵、记忆投毒、多 Agent 协议攻击、递归自我修改等 7 大类。设计了 Scanner → Assessor → Response 三级免疫 pipeline，内嵌于 Agent 的认知循环中，实现运行时检测、评估和响应，而非传统的 prompt-level guard。

**值得关注**：
- 随着 Agent 从 demo 走向生产（尤其是具身 Agent 直接操作物理世界），安全问题从"输出内容不当"升级为"行为失控"。ANIS 是第一个框架级解决方案。
- 对做 Agent 的同学：如果你的 Agent 调用工具、读写记忆、多 Agent 协作，这些 attack surface 都是真实的。ANIS 提供了可工程化的威胁模型。
- 类比：LLM 时代的 RLHF 对齐 ≈ Agent 时代的免疫系统。这是一个正在形成的新方向。

**技术启发**：能否在 VLA 模型的 action head 或 tool-calling 之前插入轻量级安全检查模块？例如，对 action proposal 做物理可行性校验，防止"把杯子扔到天花板上"这种指令被执行。
