## 1. 🤖🔄 ASPIRE：NVIDIA GEAR 提出 Agent 驱动的机器人技能自主发现系统

- **来源**：NVIDIA GEAR Lab / arXiv 2607.00272（2026-06-30 提交，HuggingFace Daily Papers 7/2 收录）
- **作者**：Runyu Lu, Yubo Wu, Letian Fu, Ajay Mandlekar, Ken Goldberg, Yuke Zhu, Linxi "Jim" Fan, Guanzhi Wang 等
- **事件**：提出 ASPIRE（Agentic Skill Programming through Iterative Robot Exploration），一个持续学习系统，让机器人以 code-as-policy 范式自主编写和优化控制程序，并将经验积累为可复用的技能库。三大核心组件：(1) 闭环执行引擎，暴露细粒度多模态 trace 用于自主故障诊断、修复合成和验证；(2) 持续扩展的技能库，将验证过的修复蒸馏为可迁移知识；(3) 进化搜索，生成多样化的任务序列和控制程序，超越单轨迹优化。
- **性能**：LIBERO-Pro 操作（扰动下）+77%，Robosuite 双手交接 +72%，BEHAVIOR-1K 长周期家务 +32%。其积累的技能库实现零样本泛化：LIBERO-Pro Long 上达 31%，此前最佳方法（含 test-time reasoning 和重试）仅 4%。
- **关注原因**：
  - **Agent + 具身 范式级结合**：这是将 Agent 的自主规划、故障诊断、经验积累能力注入机器人技能学习的系统性框架。Jim Fan（NVIDIA GEAR 负责人）此前主导了 Eureka、Voyager 等 Agent-driven 工作，ASPIRE 将这一理念推到了物理机器人层面
  - **技能库的跨 embodiment 迁移**：ASPIRE 的技能库可跨仿真和真实机器人、不同 embodiment 转移——这对于具身智能的 scaling 有根本性意义
  - **故障诊断与修复的自主化**：当前 VLA 模型执行失败后需要人工介入，ASPIRE 的闭环自主修复机制为"机器人 24/7 自主操作"提供了关键技术路径
- **技术启发**：Agent 的核心能力——感知→规划→行动→反思——正在完整地迁移到机器人领域。对我们做视频+具身的研究生来说，"视频理解 + 技能发现 + 自主修复"这个三角组合可能是下一个系统性研究方向
- **链接**：[arXiv 2607.00272](https://arxiv.org/abs/2607.00272) | [NVIDIA GEAR 项目页](https://research.nvidia.com/labs/gear/aspire/)

## 2. 🎥→🌍 World from Motion：NVIDIA 从单目视频生成动态 4D 重建

- **来源**：NVIDIA Research / arXiv 2607.01202（2026-07-01 提交）
- **作者**：Liyuan Zhu, Shengyu Huang, Amrita Mazumdar, Tianye Li, Gordon Wetzstein, Shalini De Mello, Alex Trevithick 等
- **事件**：提出 World from Motion，从单目视频无条件生成可自由渲染的动态 3D Gaussian 表示。核心创新在于：用视频扩散模型作为先验，条件化于编码了外观、几何和 3D 场景运动的稠密像素对齐渲染，来修复初始重建中的伪影并填充缺失区域。训练时构建了多视图视频对 + 动态 3DGS 的配对数据集，模拟单目重建的典型伪影。测试时将生成结果蒸馏回一致的、高质量的动态 3DGS。在 4D 重建上达到 SOTA，并能无缝泛化到具有大视角变化和动态运动的 in-the-wild 视频。
- **关注原因**：
  - **视频→3D/4D 是视频分析与具身智能的桥梁**：从视频中重建动态 3D 场景，这直接服务于机器人的环境理解和仿真构建。此前 SimFoundry 做视频→静态仿真，World from Motion 补上了"动态"这一关键缺失
  - **视频生成模型作为 4D 重建先验**：用 video diffusion model 的生成先验来"脑补"单目视频缺失的视角和运动信息——这种"生成式重建"思路正在成为视频+3D 的新范式
  - **in-the-wild 泛化**：能够处理任意视频而非受控场景，这是走向真实应用的关键一步。对视频分析方向的研究生来说，这证明了"视频理解的质量可以直接决定 3D 重建的上限"
- **技术启发**：传统视频压缩和传输基于 2D 像素，但如果能从视频中提取动态 3D 表示（4D Gaussian），那多媒体通信的"传输单元"就可以从 2D 帧变成 3D 场景——这可能是一个值得探索的交叉方向
- **链接**：[arXiv 2607.01202](https://arxiv.org/abs/2607.01202) | [NVIDIA 项目页](https://research.nvidia.com/labs/amri/projects/world-from-motion/)

## 3. 🦾📐 ABot-M0.5：统一移动与操作的"三层对齐"世界动作模型

- **来源**：arXiv 2607.00678（2026-07-01 提交，HuggingFace Daily Papers 7/2 收录）
- **作者**：Ronghan Chen, Yandan Yang 等（高德 CVLab / AMAP-CVLab）
- **事件**：提出 ABot-M0.5，一个面向移动操作（mobile manipulation）的新一代世界动作模型（WAM）。其核心洞察是：移动操作需要三个层面的对齐——**时间粒度对齐**（引入中间潜在动作桥接视频隐空间与具体控制）、**动作空间对齐**（双层级 MoE-Transformer 解耦基座移动与机械臂操作）、**训练-推理一致性对齐**（dream-forcing 训练策略，逐步在模型预测视频上训练逆动力学）。在长周期移动操作和精细控制双维度上均达到 SOTA。
- **关注原因**：
  - **世界模型从"纯预测"走向"可执行"**：传统世界模型只管预测未来帧，ABot-M0.5 的三层对齐设计让世界模型直接产出可执行动作——这是从 World Model 到 World Action Model 的关键跃迁
  - **移动+操作统一建模**：大多数 VLA/世界模型要么做导航、要么做操作，ABot-M0.5 首次系统地解决了二者在同一框架内的异构动作空间冲突问题——这对通用家用机器人至关重要
  - **dream-forcing 训练策略**：在模型自己预测的视频上训练逆动力学，解决 train-test mismatch——这个思路简洁优雅，且有理论深度
- **技术启发**：三层对齐的设计哲学——时间粒度、动作空间、训练推理一致性——不仅适用于机器人，也许可以迁移到视频生成模型的时序控制问题（例如精确控制生成视频中的动作时序和类型）
- **链接**：[arXiv 2607.00678](https://arxiv.org/abs/2607.00678) | [代码](https://github.com/amap-cvlab/ABot-Manipulation) | [HuggingFace](https://huggingface.co/papers/2607.00678)

## 4. 🧠📝 AutoMem：Stanford 让 Agent 把记忆当作可学习的认知技能

- **来源**：Stanford / arXiv 2607.01224（2026-07-01 提交）
- **作者**：Shengguang Wu, Hao Zhu, Yuhui Zhang, Xiaohan Wang, Serena Yeung-Levy（Stanford）
- **事件**：提出 AutoMem，重新定义 Agent 记忆——不把记忆当作静态的外部存储（如 RAG/向量数据库），而是作为一种**可训练的认知技能**（类比认知科学中的 metamemory）。核心机制：双层自动化循环——(1) 强 LLM 审查完整 agent trajectory，迭代优化记忆结构（prompt、文件 schema、动作词汇）；(2) 从大量 episode 中识别 agent 的"好记忆决策"，用作训练信号直接微调模型。文件系统操作被提升为与任务动作同级的"一等动作"，agent 自主决定何时读写。
- **结果**：仅优化记忆（不改任务行为），32B 开源模型性能提升 2-4 倍，**在长周期游戏中达到与 Claude Opus 4.5 和 Gemini 3.1 Pro Thinking 相当的水平**。在 Crafter、MiniHack、NetHack 三个程序化生成的长周期游戏上验证。
- **关注原因**：
  - **记忆从"架构设计"变成"训练目标"**：这是 Agent 记忆研究的范式级转变。此前大家都在设计更好的记忆架构（层次化、总结式、向量检索……），AutoMem 说：别设计了，让模型自己学会怎么记
  - **Stanford Serena Yeung 组出品质量保障**：该组在计算机视觉和 AI 领域有持续高水平产出，这个工作延续了"让模型学会学习"的研究传统
  - **与具身+视频的潜在连接**：如果把视频世界模型的记忆也看作一种可学习的技能（类比 MemLearner），那么 AutoMem 的"双循环优化"框架或许可以直接迁移到视频世界模型的长时记忆学习中
- **技术启发**："记忆不是存数据，而是学会什么时候该记、该记什么、该怎么检索"。Agent 记忆和视频世界模型记忆可能共享相同的可学习元记忆机制——这是一个被严重低估的研究交叉方向
- **链接**：[arXiv 2607.01224](https://arxiv.org/abs/2607.01224) | [项目页](https://autolearnmem.github.io/)

## 5. 🎬💰 TwelveLabs 获 1 亿美元融资 + AWS 深度绑定：视频 AI 进入产业化快车道

- **来源**：GlobeNewswire / TechStartups（2026-07-02 报道）
- **事件**：视频 AI 公司 TwelveLabs 完成 1 亿美元 B 轮融资，并宣布与 Amazon Web Services 深度合作：AWS 成为其首选云提供商，TwelveLabs 承诺将视频工作负载优化到 AWS 自研 Trainium 芯片上。TwelveLabs 的核心产品是视频智能模型（视频搜索、摘要、分类、推理），面向企业级大规模视频库。
- **关注原因**：
  - **视频 AI 企业化是行业信号**：1 亿美元融资 + AWS 深度绑定意味着视频理解不再是实验室 demo，而是正在成为企业基础设施——对你这样的视频方向研究生来说，这意味着产业需求真实且巨大
  - **Trainium 适配**：TwelveLabs 承诺优化 AWS Trainium（而非 NVIDIA GPU），暗示视频模型的推理优化可能走向专用芯片——这与视频压缩/传输的效率优化有技术交集
  - **视频理解的下游正在成形**：搜索、摘要、分类、推理——这些是视频理解的核心能力，也是你研究可以直接瞄准的工业落地场景
- **技术启发**：视频理解从"发论文"到"拿融资"的距离正在缩短。做视频方向研究的同学应该关注工业界真正需要解决什么——视频库的语义搜索、长视频摘要、跨视频推理，这些都是明确的刚需
