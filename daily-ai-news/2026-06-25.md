## 1. 🧠 Qwen-AgentWorld：阿里通义发布首个语言世界模型，打通 Agent + 世界模型

**arXiv**: [2606.24597](https://arxiv.org/abs/2606.24597) | **团队**：阿里通义（Qwen Team） | **代码**：[github.com/QwenLM/Qwen-AgentWorld](https://github.com/QwenLM/Qwen-AgentWorld)

**简述**：阿里通义团队发布了 **Qwen-AgentWorld-35B-A3B** 和 **Qwen-AgentWorld-397B-A17B**，首个能以自然语言模拟 Agent 交互环境的语言世界模型。该模型覆盖 Web、Coding、Math、Game、GUI、Search、Tool-Use 共 **7 个领域**，基于超过 **1000 万条**真实环境交互轨迹进行三阶段训练（CPT 注入世界建模能力 → SFT 激活推理 → RL 提升仿真保真度）。同时发布了 **AgentWorldBench** 新基准。

**值得关注的原因**：

- **世界模型 + Agent 的范式级结合**：不同于传统世界模型预测像素级物理状态，AgentWorld 在"语言空间"中建模环境动态，直接面向 Agent 决策——这是世界模型从具身走向通用 Agent 的关键跨越
- **双用途设计**：既可以作为解耦的**环境模拟器**支持规模化 Agent RL 训练（效果超越仅用真实环境训练），也可以作为 **Agent 基础模型的预训练 warm-up**（在 7 个 Agent 基准上一致提升）
- **百万级轨迹 + 三阶段训练**：数据规模和训练策略均可复现，对 Agent 基础模型研究有直接方法论价值

**技术启发**：是否可以将语言世界模型引入具身场景，构建"语言-物理"混合世界模型，先低代价语言规划再物理验证？这可以作为视频+具身+Agent 交叉方向的一个切入点。

## 2. 🎬 video-SALMONN-R³：让视频 LLM 学会"重看、重问、重答"的 Agent 循环

**arXiv**: [2606.24477](https://arxiv.org/abs/2606.24477) | **团队**：清华/字节跳动

**简述**：提出 **video-SALMONN-R³**，首个端到端训练的视频 LLM，通过强化学习（RL）实现"先粗看→定位关键片段→高清重看→重问→重答"的 Agent 式视频理解循环。核心创新：**无需 CoT 冷启动数据标注**，直接用 RL 训练，避免了 SFT 对预训练视频理解能力的降解风险。设计了 Re-Answer（首轮直接回答后精细化修正）和 Re-Ask（重看时重新注入问题）两个关键机制，在显著低于同类方法的计算成本下取得最优性能。

**值得关注的原因**：

- **视频理解 + Agent 思维的内化**：Rewatch→Reask→Reanswer 本质上是把 Agent 的"感知-反思-再感知"循环内化进了单一模型，而非外挂多步调用
- **纯 RL 训练范式**：绕过了昂贵的 CoT 标注，对长视频分析场景（电影、监控、具身第一人称）的方法论迁移有直接启示
- **代码和数据承诺开源**：成本优势 + 可复现性使这项工作非常适合作为视频理解研究的对比基线

**技术启发**：能不能把 R³ 的设计思想用于具身场景的视频 VLA？让机器人在执行任务时学会"回头看关键帧"而非全帧率处理，这对长时域操作任务的内存效率至关重要。

## 3. 🤖 InSight：Stanford 让 VLA 模型自主学会新技能——具身 Agent 闭环的典范

**arXiv**: [2606.24884](https://arxiv.org/abs/2606.24884) | **团队**：Stanford（Mac Schwager & Jiajun Wu 组） | **项目页**：[insight-vla.github.io](https://insight-vla.github.io)

**简述**：InSight 是一个让 VLA 模型**自主获取新技能**的框架，无需任何人类演示。核心流程：(1) VLM 规划分解——将演示自动切分为可操控的动作原语（"移动到碗""抬升""倾倒"等）；(2) **VLM 引导的数据飞轮**——识别缺失的原语→VLM 提出低层控制方案→自主尝试执行→成功则自动标注并加入训练集。在方块翻转、抽屉关闭、清扫、扭转、倾倒等任务上验证，学到的原语可组合执行新的长时域任务。

**值得关注的原因**：

- **具身 Agent 的数据闭环**：这是从"被动模仿"到"主动探索+自主标注"的关键转变，与之前 HoloAgent-0 的"反思记忆"形成互补——一个解决"学什么"，一个解决"如何学"
- **原语级可操控 VLA**：将 VLA 从黑盒策略拆解为可组合的动作原语，这种模块化设计让 Agent 可以在任务规划层面操控 VLA
- **Stanford 团队的工程深度**：从 VLM 规划→自动分割→自动标注→自主尝试→成功判定→集成训练，一行代码都不需要人工写

**技术启发**：是否可以结合视频理解能力（如 EventVLA 的事件驱动记忆）来提升 VLM 对"缺失原语"的识别精度？视频分析提供更好的任务分解能力→更精准的原语识别→更高效的自主技能获取。

## 4. 🏗️ World Value Models：为什么世界模型天然适合做机器人价值评估？

**arXiv**: [2606.24742](https://arxiv.org/abs/2606.24742) | **团队**：清华/北理工

**简述**：提出 **World Value Model (WVM)**，将世界模型与时序建模能力嫁接到机器人价值评估。核心洞察：VLM 虽强于语义理解，但其视觉编码器以静态/稀疏时序预训练为主，**缺乏深度时序理解能力**；而世界模型天然擅长时序建模和未来预测，完美匹配价值函数"理解历史+预测未来"的需求。WVM 在标准基准上取得 SOTA 价值排序相关性（VOC），同时构建了 **Suboptimal-Value-Bench**——800 条次优轨迹 + 人工逐帧标注，验证 WVM 在混合质量数据上的鲁棒性。用于策略学习时，在仿真和真实机器人上均提升操作性能。

**值得关注的原因**：

- **方法论级洞察**：论文的核心贡献不仅是模型本身，更是清晰论证了"世界模型 vs VLM 做价值评估"的优劣——VLM 的时序建模短板是已知问题，但将其与价值函数需求（深度时序理解）明确关联并给出系统性解决方案，具有领域启发意义
- **Suboptimal-Value-Bench**：混质数据是具身学习的现实瓶颈，这个新基准填补了评估缺口，适合作为后续工作的标准测试集
- **三域交叉**：WVM 是"世界模型 + 价值学习 + 机器人操作"的结合，对视频+具身+RL 方向的研究生有直接参考价值

**技术启发**：能否借鉴 WVM 的设计思路，将视频生成模型（而非传统世界模型）的时序理解能力用于 VLA 的价值评估？视频扩散模型的时序建模是否优于自回归世界模型，值得实验验证。

## 5. 🌐 Google 发布 Gemini 3.5 Flash Computer Use 公开预览：Agent 操作系统级基础设施

**来源**：[Google AI 官方更新日志](https://ai.google.dev/gemini-api/docs/changelog) | **日期**：2026 年 6 月 24 日

**简述**：Google 正式在 Gemini 3.5 Flash 上推出 **Computer Use** 工具的公开预览版。核心能力：(1) **意图驱动操作**——用高层意图而非底层指令描述操作目标；(2) **三环境原生支持**——浏览器 + 移动端 + 桌面端统一接口；(3) **可配置安全策略**；(4) **高级提示注入检测**。这意味着 Gemini 模型可以像人类一样在真实 GUI 环境中执行操作，而非仅生成操作指令。

**值得关注的原因**：

- **Agent 基础设施的标准化信号**：Google 将 Computer Use 作为模型原生能力发布（而非外挂工具），标志着 Agent 操作系统级交互正在从"研究原型"走向"API 产品"
- **对视频+具身+Agent 的间接推动**：GUI Agent 的成熟会带动三个方向的进展——(a) 视频 GUI 理解数据集需求增长；(b) 具身操控的模拟-GUI 迁移可能受益于标准化交互接口；(c) Agent 自主操作计算机做科研实验的场景更近一步
- **安全机制值得关注**：可配置安全策略 + 提示注入检测是 Agent 安全研究中可复用的设计模式

**技术启发**：能否用 Gemini Computer Use 作为具身仿真器的"数字操作员"，自动在 MuJoCo/Isaac Sim 中执行参数扫描和实验配置？视频+Agent+仿真的循环自动化实验值得探索。
