## 1. 🎯 GenCeption：视频生成模型就是通用视觉模型——DeepMind × Kaiming He，ECCV 2026

**来源：** Google DeepMind | arXiv:2607.09024 | 2026-07-10 | ECCV 2026 接收

**事件简述：**
DeepMind 发布 GenCeption，提出一个根本性命题：**大规模文本到视频生成（text-to-video generation）是计算机视觉的"next-token prediction"等价物**——即通往通用视觉智能所需的基础预训练范式。GenCeption 基于一个预训练的视频扩散模型（Genie backbone），定义了前馈感知模型，通过文本指令驱动完成多种视觉任务。在深度估计、表面法线、相机位姿估计、指代表达分割、3D关键点预测等多个任务上达到或超过专用模型（DepthAnything3、SAM3、D4RT、VGGT-Omega、Sapiens、David、Genmo、Lotus-2），且**仅需 1/7 到 1/500 的训练数据量**。此外，GenCeption 展示了一个惊人的涌现能力：仅在合成人类视频上训练的模型，能够**泛化到真实世界的动物和机器人场景**。作者包括何恺明（Kaiming He）、Andrew Zisserman 等重量级研究者。

**值得关注的原因：**
- **范式级贡献**：继 Gen4U（上周同组工作）证明视频扩散模型可作为通用编码器后，GenCeption 更进一步，直接主张视频生成预训练就是视觉的 generalist foundation path。这个命题如果成立，将从根本上改变 CV 领域的基础设施选择。
- **数据效率的极致**：7-500 倍的数据效率优势（vs 专用模型）意味着未来可能不需要为每个视觉任务收集海量标注数据，一个足够好的视频生成模型就能覆盖大量下游感知需求。
- **对具身智能的启发**：论文特别展示了在机器人场景上的泛化能力——如果视频生成预训练能理解物理世界的视觉语义，那么它成为**具身世界模型**的视觉 backbone 就非常自然。视频+具身+生成的统一趋势更加明朗。

**技术启发：**
> 视频生成 → 通用视觉感知 → 具身世界模型的路径正在打通。是否可以设计一个"生成-感知-规划"三合一的统一架构，让同一个视频扩散模型同时承担"理解世界"和"预测未来"两个职责？

🔗 [论文: arxiv.org/abs/2607.09024](https://arxiv.org/abs/2607.09024) | [项目页: genception.github.io](https://genception.github.io)

## 2. 🤖 NVIDIA GR00T 1.7 × HuggingFace LeRobot：开源具身智能生态闭环正式打通

**来源：** NVIDIA × HuggingFace | 2026-07-07 正式宣布，本周持续发酵 | Apache 2.0

**事件简述：**
NVIDIA 与 Hugging Face 宣布将 **Isaac GR00T 1.7 VLA 模型**和 **Isaac Teleop 数据采集框架**正式集成至 Hugging Face 的开源机器人库 LeRobot，并计划后续接入物理 AI 世界模型 Cosmos 3。GR00T 1.7 是首个开源可商用（Apache 2.0）的人形机器人基础 VLA，亮点包括：(1) 新 VLM backbone Cosmos-Reason2-2B（基于 Qwen3-VL），替代原有 Eagle backbone；(2) 约 32,000 小时真人演示 + 约 8,000 小时仿真预训练；(3) 在 DROID-F6（六步多阶段任务）上相比 N1.6 提升 +61%，DROID-F0 +10%；(4) 端到端 ONNX/TensorRT 导出，目标部署平台 Jetson Thor。Isaac Teleop 提供了从 XR 头显、动捕手套到 LeRobot Parquet 格式的完整数据采集管线。双方将 NVIDIA 的 300 万机器人开发者与 Hugging Face 的 1600 万 AI 开发者连接起来。

**值得关注的原因：**
- **生态闭环**：这是首次有完整的"采集→训练→评估→部署"开源管线，覆盖了具身智能开发的全生命周期。对于研究生和中小团队，意味着不必重复造轮子，可以在标准化基座上开展研究。
- **Apache 2.0 的意义**：之前具身基础模型通常受限于非商业许可或半开放，GR00T 1.7 的完全开放许可降低了产学研转换门槛。你可以直接基于它做 post-training 适配自己的机器人形态和任务。
- **世界模型本周到齐**：Cosmos 3 即将接入意味着"VLA + 世界模型"在同一生态内闭环，这对做感知-行动闭环研究的同学是重要信号。

**技术启发：**
> 开源 VLA + LeRobot 生态意味着"具身模型的快速微调与评估"成本急剧降低。可以考虑设计一个新的视频分析驱动的 VLA 评测维度（如时空理解、因果推理），直接在 LeRobot 上运行实验。

## 3. 🧠 Cortex：双向对齐的具身 Agent 框架——上海 AI Lab + 清华，长程操作新方案

**来源：** Shanghai AI Lab（上海人工智能实验室）+ 清华大学 | arXiv:2607.05377 | 2026-07-06

**事件简述：**
提出 Cortex 框架，解决当前 VLA 模型在长程操作中的核心矛盾——高层 VLM 规划与底层 VLA 执行之间的"语义-运动学鸿沟"。核心设计：(1) 将操作子任务标准化为 **32 个规范技能基元（canonical skill primitives）**，作为 VLM 与 VLA 之间的可执行接口；(2) 在数据生成管线中注入"可操作性"原则（代表性物体属性、轨迹可达性），实现对 **4,000+ 小时开源视频数据**的自动标注 + 30 小时仿真数据生成；(3) 提出 **事件均衡采样策略**，缓解子任务转换时的规划歧义；(4) 精心设计的 harness engineering，从任务上下文到技能约束的推理桥梁。Cortex 的通用 VLM 在组合微调 VLA 后，能**零样本完成未见过的真实世界长程任务**——如多阶段化学实验，这是纯 VLA 微调无法实现的。在 Libero-long 上超过单模型基线 +3.1%，RoboTwin 上 +4.1%。

**值得关注的原因：**
- **解耦与对齐并举**：Cortex 不是又一个"更大的 VLA"，而是从架构设计层面解决了 VLM-VLA 规划-执行对齐问题。这个解耦思路比追求单一巨型模型更务实，也更适合研究生在有限算力下做研究。
- **视频数据自动标注**：利用 VLM 对 4000+ 小时开放视频进行自动技能标注，说明"视频→技能图谱"的自动构建这个方向是可操作的——对做视频分析的同学是直接可借鉴的方法。
- **Skill Primitives 的设计哲学**：32 个基元不是随意选的，而是有"可操作性"约束的。这种"从真实操作需求反推表示空间"的方法论，比单纯的数据驱动更有启发性。

**技术启发：**
> Cortex 的 skill primitives 接口设计 + 大规模视频自动标注管线，能否推广到视频分析中的"时序动作定位"任务？用类似方法为视频数据集构建技能图谱。

🔗 [论文: arxiv.org/abs/2607.05377](https://arxiv.org/abs/2607.05377) | [项目页: steinate.github.io/cortex](https://steinate.github.io/cortex.github.io/)

## 4. 📊 Long-Horizon-Terminal-Bench：腾讯混元的长程 Agent 极限测试基准

**来源：** 腾讯混元（Tencent Hunyuan）| arXiv:2607.08964 | 2026-07-09

**事件简述：**
腾讯混元发布 Long-Horizon-Terminal-Bench，一个专门测试 AI Agent 在长程终端任务上极限能力的基准。包含 **46 个任务、9 个类别**（实验复现、软件工程、多模态分析、交互游戏、科学计算等）。与之前的 Terminal-Bench 不同，该基准将每个任务**分解为细粒度子任务**，提供**密集中间奖励（dense intermediate rewards）**，不仅评估 Agent 是否完成任务，还衡量它走了多远。每个任务平均需要数小时执行、数百个 episode。对 15 个前沿模型的评估结果令人震惊：**最强模型仅 15.2% pass@1（95% partial reward），完美完成的 pass@1 仅 10.9%**，所有模型的平均水平更是低至 4.3% 和 1.7%。Agent 平均消耗 990 万 token、231 个 episode、85.3 分钟执行时间。

**值得关注的原因：**
- **Agent 领域的"困难模式"基准**：当前大多数 Agent benchmark 停留在单轮问答或短任务，Long-Horizon-Terminal-Bench 填补了"真正长程、需要迭代调试"的评测空白。如果你做 Agent 研究，这个基准的引入会让实验设计更有说服力。
- **Dense Reward 的评测哲学**：传统"最终结果对=1，错=0"的 sparse reward 忽略了 Agent 的部分进度——而 Intermediate reward 能更精细地分析 Agent 的失败模式，这也是 RL 中经典思想的迁移。
- **10.9% 的结果说明什么**：即使是 GPT-5 级别的最强模型，在需要数小时执行的复杂终端任务中仍然远未可靠。这为长程 Agent 研究留下了巨大空间——工具使用、错误恢复、状态记忆等子问题都有待攻克。

**技术启发：**
> 能否将 dense reward 评测思想引入视频 Agent benchmark？例如，设计"监控视频长程事件搜索"的任务，用找到 N/全部事件的进度作为 intermediate reward。

🔗 [论文: arxiv.org/abs/2607.08964](https://arxiv.org/abs/2607.08964)

## 5. 💾 Remember When It Matters：Meta 提出主动记忆 Agent——让长程 Agent 在需要时"记起来"

**来源：** Meta Research | arXiv:2607.08716 | 2026-07-09

**事件简述：**
Meta 针对长程 Agent 的"行为状态衰减（behavioral state decay）"问题——随着执行推进，关键的上下文（任务要求、环境事实、历史尝试、未完成子目标）逐渐被挤出 context window，导致决策质量下降——提出了一个**主动记忆干预（proactive memory intervention）**方案。核心设计：一个独立的 **Memory Agent** 与 Action Agent 并行运行，维护结构化记忆库（memory bank），监控最近轨迹后**自主决定是否注入记忆提示以及注入什么内容**——关键是"沉默的权利"（remain silent when not needed）。该方案是 plug-and-play 的，可挂接到任意现有 Action Agent 上。在 Terminal-Bench 2.0 上提升 +8.3pp，τ2-Bench 上 +6.8pp。消融实验表明，选择性干预优于被动暴露记忆库、始终注入、仅 advisor、通用检索等所有基线方案。团队进一步用 SFT + GRPO 训练 Qwen3.5-27B 的 Memory Policy，实现了从 SETA 到 Terminal-Bench 的部分迁移。

**值得关注的原因：**
- **记忆机制的范式创新**："主动沉默"是一个非常优雅的设计选择——不是越频繁干预越好，而是"仅在真正需要时提醒"。这一点在认知科学上也可类比人类的工作记忆机制。
- **与视频 Agent 的结合潜力**：长视频分析 Agent（如数小时监控视频的事件检索）天然面临"信息量过大、context window 不足"的困境——这个主动记忆架构可以直接借鉴。
- **SFT + GRPO 训练记忆策略**：Memory Policy 的训练范式本身就是一个研究子方向——如何让 Agent 学会"何时该回忆、回忆什么"是一个尚未被充分探索的问题。

**技术启发：**
> "主动沉默"的记忆设计可以迁移到视频理解 Agent：在处理长视频时，让 Agent 学会自动判断哪些帧/片段需要存入记忆，哪些可以丢弃——这是主动记忆+视频压缩的结合点。

🔗 [论文: arxiv.org/abs/2607.08716](https://arxiv.org/abs/2607.08716)
