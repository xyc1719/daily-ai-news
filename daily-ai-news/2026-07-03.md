## 1. 🧠 Claude Sonnet 5：Anthropic 发布"最 Agent 的 Sonnet"

- **来源**：Anthropic 官方发布（2026-06-30）
- **事件**：Anthropic 正式发布 Claude Sonnet 5，定位为 Sonnet 系列中 Agent 能力最强的模型。具备自主规划、浏览器与终端工具调用、长时间自主运行能力。性能数据：SWE-bench Verified 63.2%（+12pt vs Opus 4.6）、SWE-bench Extended 92.4%、OSWorld-Verified 81.2%（超越人类基线 73.8%）。同时以远低于 Opus 4.8 的价格提供接近其水平的 Agent 能力。
- **关注原因**：
  - 这是"Agent 平民化"的关键节点——以 Sonnet 级别成本实现 Opus 级别的 Agent 性能，将大幅降低 AI Agent 产品的部署门槛
  - **对 Agent 研究者**：81.2% OSWorld（带工具交互的桌面代理任务）意味着 Agent 已经可以在真实 GUI 环境中持续运行——这为研究 Agent 的长期规划、错误恢复、多工具编排等提供了高质量实验平台
  - 自主使用终端/浏览器能力+长上下文（1M token）为多步工具调用 Agent 研究提供了全新的 baseline
- **技术启发**：Agent 能力的提升似乎不再依赖模型规模竞赛，而是在于"做减法"——更高效的推理+更精准的工具使用。这对我们的 Agent 架构研究可能意味着：与其做大而全的 planner，不如做精准的 skill routing。
- **链接**：[Anthropic 官方博客](https://www.anthropic.com/news/claude-sonnet-5)

## 2. 🎥🤖 DVG-WM：解耦式视频生成高效具身世界模型

- **来源**：arXiv 2606.32028（2026-07-01 提交）
- **事件**：提出 Disentangled Video Generation World Model (DVG-WM)，其核心创新在于将视频世界模型的动力学预测与视觉像素渲染解耦——先在低维特征空间预测场景动态变化，再单独合成高质量视觉效果。这解决了当前视频世界模型"速度慢则细节差、速度快则质量低"的根本性困境。
- **关注原因**：
  - **视频+具身+世界模型 三域交叉的完美案例**：用视频生成模型做具身规划，同时用解耦架构提升推理效率
  - 对做具身世界模型的研究生有直接参考价值：传统 video world model 用于 iterative planning 时推理速度是瓶颈，DVG-WM 的解耦思路可能改变这个游戏的规则
  - 架构上将"世界动力学建模"和"视频渲染"分离，这种设计哲学可迁移到其他生成式世界模型任务中
- **技术启发**：视频世界模型不一定要"端到端"——把物理推演和视觉渲染解耦，也许是一种更好的 scaling 方式。是否可以把这个思路应用到 VLA 训练数据的合成中（先推演轨迹，再渲染视频）？
- **链接**：[arXiv 2606.32028](https://arxiv.org/abs/2606.32028) | [HuggingFace](https://huggingface.co/papers/2606.32028)

## 3. 🧍→🤖 Human-as-Humanoid：从人类视频零样本构造人形机器人技能

- **来源**：arXiv 2606.32009（2026-07-01 提交）
- **事件**：提出 Human-as-Humanoid 框架，通过"人类对齐的具身映射"将人类的第一/第三人称（ego-exo）视频直接转化为高自由度人形机器人的可执行动作。核心创新在于不依赖遥操作（teleoperation）演示数据，而是从海量互联网人类视频中提取技能。在数据收集效率上达到了遥操作方法的 4.8-7.2 倍吞吐量增益。
- **关注原因**：
  - **视频→具身的范式级突破**：如果人形机器人可以从 YouTube 视频学习技能，那数据瓶颈将大幅缓解。这对整个机器人学习领域的 scaling law 有根本性影响
  - 该框架解决了高自由度（high-DoF）人形机器人的动作映射难题——人类和机器人的运动学结构差异是长期未解决的问题
  - 对"视频分析+具身智能"交叉方向的研究生：这可能是"视频理解如何赋能机器人"的教科书级案例
- **技术启发**：视频数据的价值可能远超当前认知——不仅用于理解，还能直接转化为机器人的执行策略。我们的视频理解工作是否也能"向下游延伸"到具身执行？
- **链接**：[arXiv 2606.32009](https://arxiv.org/abs/2606.32009) | [HuggingFace](https://huggingface.co/papers/2606.32009)

## 4. 🦾 Play2Perfect：灵巧操作从"会玩"到"会装配"

- **来源**：Stanford / arXiv 2606.26428（2026-06-25 提交，近日热度上升）
- **事件**：来自 Stanford 团队，研究多指灵巧手执行精密装配任务的预训练方法。核心思想"先玩后精"（Play then Perfect）：先用多样化的目标物体做任务无关的游戏式探索（play），再在精密装配任务上微调（perfect）。系统性地研究了 play 阶段的哪些因素（目标多样性、探索策略、奖励设计）对后续装配成功影响最大。
- **关注原因**：
  - **灵巧操作领域的系统性研究**：大多数灵巧操作工作都在单一任务上刷指标，Play2Perfect 系统地研究了预训练到微调的转移规律——这种"方法论级"的工作对研究生更有参考价值
  - Play→Perfect 的两阶段训练范式与 NLP 领域的 pre-train→fine-tune 哲学一致，暗示了机器人学习的通用范式可能正在浮现
  - 精密装配是制造业的"圣杯"任务，该方向一旦突破将有巨大的实际应用价值
- **技术启发**：具身智能的训练或许也可以借鉴 NLP 的"预训练+指令微调"范式——先用多样化的交互数据学"世界常识"，再用少量高质量任务数据学"执行精度"
- **链接**：[arXiv 2606.26428](https://arxiv.org/abs/2606.26428) | [项目主页](https://play2perfect.github.io/)

## 5. 🎞️🧠 MemLearner：让视频世界模型学会"记住该记住的"

- **来源**：arXiv 2606.31734（2026-07-01 提交，HuggingFace Daily Papers 收录）
- **事件**：提出 MemLearner，一种用于视频世界模型的上下文记忆机制。其核心创新在于不训练额外的记忆模块，而是让视频生成模型本身学习"从历史帧中查询（query）对未来预测有用的上下文"。相比传统的 rule-based 或 fixed-window 上下文检索方法，MemLearner 能自适应地决定记住什么、忘记什么。
- **关注原因**：
  - **视频世界模型+记忆机制**：这是视频生成和 Agent 记忆研究的交汇点。传统世界模型在长时预测中会遇到上下文遗忘问题，MemLearner 的方案优雅且可扩展
  - 与 Agent 的记忆研究方向高度呼应——Agent 需要记住过去的状态和动作，视频世界模型需要记住过去的帧，两种记忆机制可能共享设计空间
  - 利用预训练视频生成模型本身的先验知识（而非从头训练记忆模块），这种"复用已有能力"的思路值得借鉴
- **技术启发**：记忆不是"存数据"，而是"学会查询"。视频世界模型的记忆和 Agent 的记忆可能共享相同的核心原理——这或许是一个被低估的研究交叉点
- **链接**：[arXiv 2606.31734](https://arxiv.org/abs/2606.31734) | [HuggingFace](https://huggingface.co/papers/2606.31734)
